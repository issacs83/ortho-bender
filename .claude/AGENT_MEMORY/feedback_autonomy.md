# 자율 진행 위임 (2026-05-01)

## 사용자 지시 (원문)
> "중요한 포인트는 모든 권한을 부여하니 알아서 진행되도록 준비해요"

## 규칙
**기본값: 자율 진행. 매 단계 확인 안 받음.**

### 자율 진행 OK
- 정해진 작업의 실행 (계획된 Phase, 명확한 태스크)
- 합리적 판단 가능한 기술 결정 (라이브러리 선택, 파일 구조 등)
- 빌드/테스트/디버그 사이클
- 코드 작성 / 수정 / 리팩터링
- 문서 작성

### 묻고 진행 (자율 금지)
- 비즈니스 판단 / 비용 / 시간 (PSU 업그레이드, 부품 주문 등)
- 외부 시스템에 비가역 변경 (eMMC flash, deploy, force push to main)
- Phase Gate (CLAUDE.md HARD GATE 결재 규칙)
- 사용자 명시 지시와 충돌하는 결정

### Why
사용자는 매번 OK 받느라 답답함을 겪음. 결정사항/플랜이 메모리에 다 있는데도 또 묻는 게 비효율적.
"빨리 진행하고 결과로 보여줘" 모드. 단, 진짜 큰 결정은 묻기.

### How to apply
1. 메모리 먼저 읽고 결정사항 확인
2. 안 정해진 사소한 거는 합리적 default로 진행 (사유 메모/커밋 메시지에 명시)
3. 정해진 큰 결정 (위 "묻고 진행" 카테고리)만 사용자에게 보고
4. 진행 중 보고는 dashboard agent_progress 이벤트 / 텔레그램으로 자동
5. milestone 도달 시 결재 요청 (Phase 완료 등)
